  • Data representation
    • The problem
      • Different machines, operating systems, languages use different formats for representating data
        • big endian/ little endian
        • floating point format
        • ASCII/EBCDIC
        • different size of integers
        • different memory layout/size of data structures
      • Need a machine indepent method for describing data structure
    • Presentation context
      • abstract syntax: describes data structure
      • transfer syntax: rules for encoding/decoding data structures
    • Abstract Syntax Notation 1 (ASN.1)
      • a data description language
      • Primitive types

        • Tag

        		 Type
        1 BOOLEAN
        2 INTEGER
        3 BITSTRING
        4 OCTETSTRING
        5 NULL
        6 OBJECT ID
        9 REAL
        10 ENUMERATED
      • Composed types
          • Tag Type Comments
          16 SEQUENCE ordered list
          16 SEQUENCE OF ordered list of the same type
          17 SET unordered list
          17 SET OF unordered list of the same type
          11 CHOICE like union in C
      • Predefined composed types
        • NUMERIC STRING, UTC TIME, GRAPHICS STRING, ...
      • Tag
        • Each data type can have a tag type and a tag value
          • UNIVERSAL: Predefined types
          • APPLICATION: Data types used by OSI protocols
          • PRIVATE: user defined data types
          • Context specific: identify the position of element (e.g., used to indetify different commands in SNMP)
    • Basic Encoding Rules (BER)
      • describes how each data type are to be encoded/decoded to/from a bit sequence
      • Encoding rules
        • Encoded with TAG+LENGTH+VALUE
        • TAG: 8 bits (tag type, primitive/constructed, tag number)
        • LENGTH: length of encoded data
        • VALUE: encoded data
      • BER example
  • Data Compression
    • Encoding a finite set of equally likely symbols
    • Frequency dependent coding
      • Huffman coding
    • Context dependent encoding
      • Run length coding
      • Lempel-Ziv compression algorithm
      • Video compression technologies (H.261, JPEG, MPEG)
  • Network security
    • Data encryption (cryptography)

      • Data Encryption Standard (DES)
        • Key distribution problem
      • Public key cryptography
        • RSA algorithm
    • Authentication
      • Proving who you are
      • Authentication using public key
