The Session Layer

  • Introduction
    • Services provided to upper layer
      • Connection management
        • connection establishment
        • connection release
      • Structuring data transfer
      • Synchronization
      • Dialogue control
    • Services required from lower layer
      • Reliable transport connection
      • Transport connection carries one session at a time
  • Connection management
    • Session vs. connection
      • one-to-one mapping
      • many-to-one mapping
        • Sessions are established and released very frequently
        • Keeps a long-live connection to serve come-and-go sessions
      • one-to-many
        • For some reasons (security, fault tolent, ...), one session may span multiple connections
    • 3-way handshaking for connection establishment and release
      • S-CONNECT
        • .request, .indication, .response, .confirm
      • S-RELEASE
        • .request, .indication, .response, .confirm
      • S-U-ABORT (ungraceful close)
        • .request, .indication
  • Structing data transfer
    • Activities: groups of semantically-related messages
      • For example, a file in ftping
      • Put markers in data stream
        • sender: S-ACTIVITY-START.request, S-ACTIVITY-END.request
        • receiver: S-ACTIVITY-START.indication, S-ACTIVITY-END.indication
      • An activity can be interrupted/resumed
        • S-ACTIVITY-INTERRUPT.request, S-ACTIVITY-RESUME.request
      • Only one entity can interrupt activity
        • Use token to control who can interrupt
        • S-TOKEN-GIVE, S-TOKEN-PLEASE
  • Synchronization and recovery
    • Unforeseen errors can occur
      • network down
      • application temporarily unable to process/receive data
    • Add check (synchronization) points into data stream to help recover
      • S-SYNC-MAJOR, S-SYNC-MINOR, S-RESYNCHRONIZE
        • .request, .confirm
        • SYNC must be confirmed before sending any data
    • It is application's responsibility to implement rollback/recovery
  • Dialogue Management
    • To insure that at most one entity is sending data (teleconferencing)
    • Enforce half-duplex mode by data-token
      • S-TOKEN-GIVE, S-TOKEN-PLEASE
    • Allows expedited data transfer (14 bytes)
