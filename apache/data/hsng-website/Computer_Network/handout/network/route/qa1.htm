廣播是將一封包送到網路上所有的節點。可使用的方法有：

  1. 從 source 到其他每一節點分別送一份封包
    缺點：浪費頻寬，需要知道網路上所有的節點位址
  2. Flooding：每一個收到封包的節點均將此封包轉送給其
    　　　　 他所有相鄰的節點
    缺點：一個節點可能收到好幾份相同的封包。造成頻
    　　　寬浪費。而且如何停止 flooding 也是一個問題。
  3. Reverse Path Forwarding (RDF)：source 先將封包送到
    所有的鄰居。然後其他的節點在收到封包後，如果這
    封包是由此節點到 source 的最短路徑上的節點送來的
    ，則再轉送給其他所有相鄰的節點，否則就將此封包
