Routing and Multicast

  • Theory of routing
    • Action of routing
      • Simply look up packet id and forward to appropriate
        outgoing link
    • Question: how does network determine and maintain routing

    • Goals of routing
      • Scalability
      • Safe internection of heterogeneous networks
      • Robust, fault-tolent, self-healing
      • Support QOS
      • Good performance (high utilization, low delay, ...)
    • Types of routing algorithms
      • centralized vs decentralized
      • static, dynamic, or adaptive
    • Link-State routing (LS)
      • Routing with global informatio: link cost
      • Quasi-centralized: broadcast link costs perioidcally or
        when changed
      • Use Dijkstra or Bellman-Ford shortest path algorithm
        to find the best route
      • Details of Dijkstra (Bellman-Ford) algorithms omitted
    • Distance vector routing (DV)
      • Asynchronous, distributed algorithm
      • Link cost only passed to neighborhood nodes
      • Use distributed version of Bellman-Ford algorithm to find
        the best route
      • Problems
        • looping: good news travels fast, bad news travels slow
          • Split horizon
            • send infinity cost to the reverse direction
              of the shortest path
          • Poisoned reverse
            • constantly increase cost implies loops, sent
              PR updates to start hold down
          • hold down
            • do not send updates during a period of time
        • Oscillation
          • avoid periodic change
          • link costs should not be strictly increasing functions
            of traffic load
    • Comparison of LS and DV algorithms
      • Message complexity
      • Robustness
      • Speed of convergence
  • Routing protocols
    • OSI Routing
      • Hierarchical structure of networks
        • End System (ES)
        • Intradomain intermediate system (IS)
        • Interdomain intermediate system (IS)
      • ES-IS routing
        • discovery protocol: ESs and ISs learn about each other
          through hello packets
        • Hello packets: EHS (ES hello message), ISH (IS hello
          message)
        • Hello conveys network-layer(NSAP) addresses and
          subnetwork (SNPA) addresses (data-link layer)
        • Redirection (IS->ES)
        • Map to TCP/IP and ICMP
      • IS-IS routing
        • Two level hierarchy
          • Level 1
            • routing within the same area (a group of
              contiguous networks)
            • Link state routing (complete, consistent
              picture of network topology)
            • Knowledge of level 2 routers connected
              to the same area
          • Level 2
            • routing between level 1 areas within a domain
            • Link state routing
            • strict hierarchy (pass through level 2 routers
              only)
        • Link cost
          • 3 metrics: delay, expense, error rate
          • Maximum value: 64 for link, 1024 for path
        • Support of multiple network-layer protocols
          • Integrated IS-IS
          • Ships-in-the-night
      • Interdomain Routing Protocol (IDRP)
        • Routing between domains (different administrative control)
        • Border intermediate system (BIS)(exterior gateways): router
          for interdomain routing
        • Distance vector routing
          • Exchange information about id's of routing domains,
            paths to these domains
          • exclusion list, inclusion list
        • Policy dependent routing
    • Internet routing protocols
      • Intradomain routing protocols
        • Routing Information Protocol (RIP)
          • Originally designed for Xerox PARC Universal
            Protocol (used in XNS)
          • Adopted by UNIX and TCP/IP in 1982 (e.g.,
            routed of BSD)
          • RFC 1058 [1988]
          • RIPv2: RFC 1388 [1993]
          • Widely used in PC networking
            • Novell, 3Com, Apple Talk, Banyan,
              Ungerman-Bass
          • Distance vector routing
            • use hop count as cost metric (up to 15)
            • restrict size of the network to 15
          • Routing table

            • Timers
              • routing update timer (30 s): time to send
                routing table to neighbors
              • route invalid timer (90 s): time to make an
                entry old
              • route flush timer (270 s): time to remove
                an entry
          • RIP packet format
            • commands: request or reply
            • version number
            • up to 25 destination addresses
          • Stability
            • hop count limit: 15 means infinity
            • hold down:
              • Asynchronous transmission may make old
                information update new information
              • Hold down any changes that might affect
                recently removed routes for some period of time
            • Split horizons
              • no update on backward route
            • Poison Reverse Update
              • constantly increase cost implies loops
        • Interior Gateway Routing Protocol (IGRP)
          • IGRP
            • Developed by Cisco in mid-1980's for routing
              within an Autonomous system (AS)
            • Enhanced IGRP was proposed in 1990's
            • Distance vector routing
            • Combination of metrics
              • Delay (1..224)
              • Bandwidth (1200bps to 10Gbps)
              • Reliability (1..255)
              • Load (1..255)
            • Support multipath routing
              • among equal metric paths
              • among different metric paths
            • timers: update, invalid, flush, hold-down
          • Enhanced IGRP
            • New features
              • fast convergence: store neighbors' routing
                tables (Diffusing Update Algorithm, DUAL
                )
              • variable length subnet masks
              • partial, bounded updates: update only when
                a route cost change
              • multiple network-layer support: IP, Apple
                Talk, Novell
            • New technology
              • Neighbor discovery/recovery
                • Dynamically learn new neighbors
                • Send hello to check alive
              • Reliable Transport Protocol (RTP)
                • Guaranteed ordered delivery of IGRP
                  packets
              • DUAL finite state machine: compute routes
              • Protocol-dependent modules: for network layer
                • protocol specific requirements (e.g., IP
                  encapsulated IGRP packets)
        • Open Shortest Path First (OSPF)
          • RFC 1247
          • Characteristics
            • Open: not proprietary
            • SPF: Dijkstra algorithm (link state routing)
          • Routing Hierarchy
            • AS: collections of networks under a common
              admistrative (areas)
            • Area: a group of contiguous networks (e.g., LAN)
              and routers (area border routers, multiple ports)
            • Topological database: overall picture of networks
              in relationship to routers
            • Domain: network of routers with the same
              topological database (~AS)
            • In order to reduce routing traffic, topology of a
              domain is not passed to other routers outside of
              the domain
            • Two types of OSPF routing: intra-area and
              interarea (backbone)
          • SPF algorithm
            • The basis for OSPF operations
            • Routers use Hello protocol to acquire neighbors
            • If an area (network) is attached by more than
              one router, a router is elected as the designated
              router which is responsible for generating link
              state information (and another one for backup)
            • Each router compute a shortest-path tree (with
              itself as the root) from the topological database:
              Dijkstra's algorithm
          • Special features
            • Reliable broadcast for flooding link state information
            • Euqal-cost multipath routing
            • Routing based on Type of Service (TOS)
              • IP TOS: delay, throughput, reliability
              • Seperate routing table for each TOS: needs
                8 routing tables?
            • Allows level-2 routers to send path cost to level-1
              routers
            • Runs on top of IP
              • allows variable-length subnet mask
              • IS-IS uses underlying data-link control protocol
      • Interdomain routing protocols
        • Concerns
          • connecting networks under different administrative control
          • one or more routers in each domain are border gateways
            • internet jargon: exterior gateways
            • ISO jargon: border intermediate systems (BIS)
          • Border gateways form a backbone for routing
          • Policy dependent routing
            • each domain determines its own policy for how packets
              to be routed/forwarded
        • Exterior Gateway Protocol (EGP)
          • RFC 904 [1984]
          • Widely used in Internet (ARPANET) in the past, but
            now phased out by BGP, IDRP
          • Routing among AS

          • Distributes complete route updates
            • contain network reachability information (so it is
              also called a reachability protocol, not a routing
              protocol)
            • no performance metric
            • no intelligence
            • allow routing loops
          • Primitive functions
            • establish neighbors
            • poll to see if neighbors alive
            • distribute updates
          • Messages
            • Message Function
            Neighbor acquisition Establishes/de-establishes neighbors
            Neighbor reachability Determines if neighbors are alive
            Poll Determines reachability of a particular network
            Routing update Provides routing updates
            Error Indicates error conditions
        • Border Gateway Protocol (BGP)
          • RFC 1163
          • Ancestor of IDRP
          • Distance vector routing
          • Can be used within and between ASs
            • one router will be elected to represent an AS
              if more than one exist
          • Routing table keeps all feasible paths to a destination
            network
            • but advertises only the optimal path to neighbors
          • No periodic refresh of the entire table, exchanges
            only changes in tables
          • Update messages are sent over TCP for reliable
            delivery
          • Messages
            • Open: first send, reply with keepalive
            • Keepalive: send often enough to keep the
              hold-time timer from expiring
            • Update: routing updates, include AS path (list
              of ASs on the path) and next hop information
            • Notification: send when an error is detected
          • Metric: combination of different metric with the
            degree of preference

  • Multicasting
    • Broadcast routing
      • N point-to-point
      • Flooding
      • Reverse path forwarding (RPF)
    • MBONE
      • Multicast Backbone (1992)
      • Begun from audiocast of IETF
      • Terminologies
        • Islands: subnetworks that support IP multicast
        • Mrouters: routers that run multicast deamon (mrouted)
        • Tunnels: virtual point-to-point links that connect mrouters
      • A host joins/leaves a multicast by JoinHostGroup( ) and LeaveHostGroup( )
        • a join requires sending an Internet Group Management
          Protocol (IGMP) to a mrouter in the island
        • multicast packets are routed by protocols such as DVMRP
          , PIM, CBT, and MOSPF
      • Applications
        • Session Directory (SD): tools for show/join/create multicast
          sessions
        • NetVideo (NV): real time video multicasting
        • Visual Audio Tool (VAT): real time audio multicasting
        • WhiteBoard (WB): shared whiteboard
    • Mulricast protocols
      • Distance Vector Multicast Routing Protocol (DVMRP, RFC 1075)
        • distance vector algorithm
        • Reverse Path Forwarding
          • RPF is for broadcast originally
          • DVMRP modifies it to Truncated RPF
        • Horizon split with Poison reverse to avoid looping
        • IGMP to learn group members
          • there is a "subtype" field in DVMRP header indicates
            • Group membership queries: subtype=1
            • Group membership reports: subtype=2
            • DVMRP routing updates: subtype=3
          • Queries are generated periodically by mrouters (with
            address 224.0.0.1) to query group members on the
            subnetwork
          • End systems send reports to mrouters upon receiving
            queries
        • Truncated RPF
          • Duplicate messages as in RPF
          • On a subnetwork, only one mrouter is responsible for
            forwarding (dominating mrouter)
            • the one with shortest distance to the source
            • tie break by preferring lower address
          • dominating mrouter then decides if the connected
            subnetwork is a child or a leaf for a particular multicast
            tree
          • prune messages are sent to neighbor mrouters to cut the
            braodcast tree to the multicast tree
          • dynamically join by sending a graft message
      • Protocol-Independent Multicast (PIM)
        • dense mode
          • like DVMRP, use truncated RPF

        • sparse mode
          • rendezvous point (RP)
            • sender and receivers register with rendezvous
              point first (prechosen)
            • optimal multicast tree is then automatically for
              med as traffic goes on
            • there is a list of RPs to choose from
            • support RP (also called core) migration

      • Core Based Tree (CBT)
        • Core is the RP
        • organize RPs into a multilevel global hierarchy
          • transparent to senders and receivers, they all only
            contact with their designated routers (level 0 RP)
      • Multicat Open Shortest Path First (MOSPF, RFC 1584)
        • modification of OSPF to multicast
        • based on shortest path from source to each destination
