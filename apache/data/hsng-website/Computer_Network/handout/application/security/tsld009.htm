Two Fundamental Cryptographic Principles

  • All messages must contain redundancy

    • To avoid intruder sending a lot of random messages
      • For example, if a mail-order message only contain 3 bytes, one byte for order amount, two bytes for product id, then it is very likely to be able to generate valid messages randomly
    • Passive intruder may be able to capture a lot of (plaintext, ciphertext) pairs

  • Some measures must be taken to prevent intruders

    • Active intruder can listen and playback messages
    • Sequence number and timer are required in a message to reject duplicate and old messages
