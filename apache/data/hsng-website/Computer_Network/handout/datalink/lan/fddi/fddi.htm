FDDI：Fiber Distributed Data Interface

FDDI

    ASCX3T9.C標準(ANSI)

    媒介接觸控制(MAC)

    實體層通訊協定(PHY)：數位數據編碼方式

    實體層傳輸媒介相關部份(PMD)：

      光纖驅動器、接收器
      工作站與環狀網路連接
      傳輸線

    層次管理(LMT)：管理各層次工作

雙環結構

    主環

    次環

設備

    工作站

      Class A：同時與主環及次環連接

        架構交換電路可跳接雙環變單環

      Class B：只有與主環連接

    集中器(Concentrator)

      二對PHY/PMD接主環及次環
      其他PHY/PMD接class B工作站
      架構交換電路可隔離class B工作站或將雙環變單環

網路架構

    實體架構：工作站與光纖間之硬體連接方式

      樹狀雙環(Dual Ring with Trees)
      不含樹狀之雙環(Dual Ring without Trees)
      樹狀單環(Wrapped Ring with Trees)
      不含樹狀單環(Wrapped Ring without Trees)
      單一樹狀結構(A Single Tree)

    邏輯架構：Token或資料的傳遞路徑

      一合法的邏輯架構最多只能有二環：主環與次環
      若某光纖故障，將只剩下一環
      若多個不相鄰的故障發生，將形成數個不相連的單環結構

    資料速率：100 Mbps

    網路長度：200公里

    可接1000台工作站

    範例：圖9-7 , 9-8

通訊協定

    利用Token來控制資料傳送秩序

    工作站要傳資料前先取得Token，資料傳完馬上釋放Token

    資料清除由傳送者負責

    Token依邏輯架構循環於各工作站間

    工作站延遲時間：Token長度(非1-bit delay)

    沒有Free Token與Busy Token之分

資料格式

位址模式

    16-bit , 48-bit兩種

    16-bit工作站須具以下功能

      轉送具48-bit位址的資料
      辨識48-bit廣播位址(均為"1")
      對具48-bit位址的要求控制訊息及求救控制訊息做出正確反應

    48-bit工作站

      有一獨立的16-bit位址
      辨識16-bit廣播位址(均為"1")

頻寬分配(Bandwidth Allocation)

    兩個基本要求

      支援Stream Traffic與Bursty Traffic混合使用
      提供多重資料對話(Multiframe Dialogue)

    支援Stream Traffic與Bursty Traffic

      同步資料(Synchronous Traffic)：適合stream traffic
      異步資料(Asychronous Traffic)：適合bursty traffic
      分配方法：

        TTRT：Target Token Rotation Time
        分配結果必須滿足：SAi + Dmax + Fmax + Token_time ≦ TTRT
        SAi：工作站i所分配到的同步資料量，由SMT協定決定
        Dmax：propagation delay for one complete circuit of the ring
        Fmax：傳送一個最長資料(4500 bytes)所需要的時間
        Token_time：傳送一個token所需要的時間

      運作方法

        每一工作站有3個counter

          TRT：Token Rotation Timer
          THT：Token Holding Timer
          LC：Late Counter

        當一工作站拿到一Token時

          Case 1：LC = 0：Token早到(TRT > 0)

          1. 工作站可以傳同步及異步資料
          2. THT = TRT , TRT← TTRT , LC← 0
          3. 啟動TRT(即TRT開始計時)
          4. 工作站可以傳SAi時間內的同步資料
          5. 啟動THT
          6. 在THT≧0前，可以傳送異步資料
          7. 異步資料分8種priority，對每一priority i T_PR(i)為
            Token繞一圈後，仍可送priority i資料的最大時間，
            T_PR(i)≦TTRT

    多重資料對話(Multiframe Dialogue)

      當一工作站卻與令一工作站做進一步對話時，可利用限制性信物
      　(stricted token)來取得所有異步通訊頻寬
      運作方法

      1. 工作站取得無限制性信物(必須是接收最後一筆異步資料的工
        作站)
      2. 送出第一筆資料給目的地
      3. 送出限制性信物
      4. 此後其他工作站無法傳異步資料

TTR (Timed Token Rotation) Protocol特性

    允許工作站以SMT協定保留同步資料頻寬

    任何工作站連續二次拿到Token的時間絕不大於二倍的TTRT

    頻道使用率 = (TTRT - RL) / TTRT

      RL：Ring Latency，即在空載時，信物繞一圈所需時間

    TTRT的選擇

      太小：

        少量、高頻率
        低頻道使用率

      太大：

        高頻道使用率，特別是高載時

環的監督

    分散式

    任一工作站發現不正常現象，便可要求網路重建

    不正常現象：

      網路上太久沒有活動(token lost)
      不正確的活動，如資料沒有清除

    錯誤偵測與更正

      要求信物程序(Claim Token Process)
      重建(Initialization Process)
      求救(Beacon Process)

    二種MAC控制訊息

      要求信物訊息(Claim frame)
      求救訊息(Beacon frame)

要求信物程序

    任何工作站發現不正常便可送出claim frame要求進行要求信物程序

      決定TTRT值
      決定由誰重新啟動Token

    每一欲要求信物的工作站持續送出claim frames，並設定TTRT值

    每一工作站收到訊息時，可能

      停止競爭
      繼續競爭：消除收到的訊息

    仲裁原則

      送出TTRT較小的訊息有較高的優先權
      若TTRT相同，有較長的原始位址者有較高優先權(48 v.s. 16)
      位址較長有優先權

    競爭過程一直進行到某一工作站收到自己的claim frame

      其他工作站一最後的claim frame設定TTRT值

重建程序

    在要求信物程序中脫穎而出的工作站須負責重建環狀網路(稱之重建
    　工作站)

    其他工作站在收到claim process後便自動進入重建程序

    重建工作站送出一無限制信物，此信物繞第一圈時不可被capture

    其他工作站收到此信物後，便進入運作狀態(設定TRT = TTRT)

    重建工作站回收信物後，便可開始傳送同步，異步資料

求救程序

    用來隔離嚴重的網路故障，如claim process失敗

    工作站進入求救程序後，會持續送出求救訊息，直到回收求救訊息。
    　此時表示邏輯環已修復，可進行要求信物即重建程序

網路可靠度規格

    工作站跳接：故障或停機的工作站可用自動光學跳接交換電路

    集中器：集中器跳接隔離某一光纖或工作站

    雙環結構：雙環中某一工作站或光纖故障時，可重組成單環結構

資料編碼

    光纖屬於傳送類比訊號之媒介

    以ASK較容易(Amplitude Shift-keying)

    缺點：無法提供同步功能

    採用4B/5B編碼法(125 MHz)而不是Manchester (須200 Mhz)

    4B/5B：將4個位元轉成5個位元

      5位元中至少有2次轉換(2個1)
      5位元中至多有3個連續的"0"

    再用NRZI (NonReturn to Zero Inverted)編碼後才送上光纖

      1：在位元期間的開始有一轉換(transition)
      0：在位元期間的開始沒有轉換
