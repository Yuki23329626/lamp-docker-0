網路特性

    主動式訊號增益器

    單向傳送

    利用Token控制資料傳送順序

    適合即時應用系統，可提供延遲保證

訊號增益器功能

    加入資料訊息(Frame Insertion)：需先拿到Token

    接收資料訊息(Frame Reception)：判斷目的地位址

    消除資料訊息(Frame Removal)：

      由sender消除：receiver可加入ACK
      由receiver消除：可提高效率

訊號增益器狀態

    聆聽狀態

      特殊字元辨識(Token, Address等)
      將資料拷貝給工作站
      修改一位元資料
      會造成一位元延遲

    傳送狀態

      將資料傳送出去並從另一端接收

    跳接狀態

      工作站故障或關機

環狀網路優點

    在高負載下可有高的使用率

    點對點通訊

      訊號可在每一點增強
      較低傳送錯誤率
      基頻傳輸時，可有較長距離
      可採光纖線路

    容易找出故障線路

環狀網路缺點

    線路斷裂或增益器故障會造成網路癱瘓

    安裝複雜

環狀網路種類

    槽式環狀網路(Slotted Ring)

      將網路位元長度(即網路上可以同時存在的位元數)分割成若干個
      　固定長度的槽
      每個槽有以下欄位

        狀態位元(busy or empty)
        目的地位址
        原始點位址
        資料欄
        控制欄：資料傳送結果

      工作站利用空槽傳資料
      資料必須切割成固定大小
      資料必須由sender消除並將該槽釋放
      缺點：二個位址欄，浪費頻寬

    暫存器插入式環狀網路

      訊號增益器有一個位移暫存器及一個緩衝器
      接收資料

        等目的地位址進入位移暫存器候，辨識是否為本站資料
        是則將資料轉給工作站並丟棄位移暫存器內之位址
        不是則將資料依序移入/出暫存器

      若要傳資料，須先移入緩衝器
      當一筆資料從位移暫存器中完全移出後，假如下面條件符合，
      　此工作站便可傳資料。

        有一筆資料在緩衝器中待傳
        位移暫存器有足夠空位元位置

      公平性

        在輕負載時，資料很快就可以送出
        在高負載時，每一工作站一次只能傳一筆資料

      高線路使用率
      資料長度可不固定
      允許多筆資料同時在網路上傳送
      較長傳送時間

    信物環狀網路(Token Ring)

      Free Token：一特殊位元字串，如11111111
      Busy Token：與Free Token只差最後一位元，如11111110
      當一工作站收到Free Token時，可以將它改成Busy Token並
      　將資料隨後送出
      資料送完後須再送出一Free Token
      網路上可能有多個Free Tokens

        為避免複雜，送端只在接收到Busy Token回來後，才
        　將Free Token送出

      資料可由送端或接收端消除
      容易送回應訊息
      公平
      高負載時，線路使用率高
      須維護信物

IBM Token Ring

    星－環結構

    主要設備

      集中器(Ring Wiring Concentrator, RWC)

        以線接器(wiring lobe)接工作站
        線接器跳接(lobe-bypass)

          隔離障礙
          網路維護與重組

      網路橋(Bridge)

        高速數位交換技術
        連接數個環狀網路成一大網路
        功能

          過濾
          輸出/入緩衝
          Routing
          傳送資料

      網路閘門(Gateway)

        連接區域網路與長程網路
        功能

      位址轉換
      速率與協定轉換
      資料切割與重組

    資料格式

      MAC Frame = HEADER + LLC-Frame + TRAILER
      HEADER = STARTING DELIMITER + PCF + DA + SA
      TRAILER = FCS(CRC) + ENDING DELIMITER + PCF
      STARTING PCF (Physical Control Field)

        PM (Priority Mode)：資料priority
        PR (Priority Reservation)：工作站預約的最高priority
        T (Token)：Free or Busy
        MC (Monitor Count)：一般資料或控制資料
        RR (Reserved Bits)
        Control indicator：資料狀態或網管訊息用

      Ending PCF

        ARI (Address Recognized Indicator)：目的地工作站設
        　定已拷貝此資料
        FCI (Frame Copied Indicator)：目的地工作站是否已拷
        　貝此資料
        SEI (Soft Error Indicator)：Software Error
        ARI, FCI均有2份，以便檢查

    網路運作原理

      每個工作站得到Free Token後，將它改為Busy Token並送
      　出資料
      單一信物：送端回收Busy Token後才送出Free Token
      資料由送端消除
      回應：

        接收端設定ARI表示存在，FCI表示已拷貝
        ARI=1, FCI=0表示接收端有狀況

    信物監督者

      負責監督信物流失與不斷循環繞之忙碌信物之處理
      信物流失

        原因：干擾或雜訊
        偵測：信物監督者每次送出一信物時，便設定一timer
        　。如時間到了卻沒有再收到信物，表示已流失
        處理：產生新的自由信物

      不斷環繞之忙碌信物

        原因：傳送端因故當機
        偵測：當信物經過監督者時，Monitor Count Flag會被
        　設定。新信物送出時，送端則會reset成0。所以如果監
        　督者收到MC=1之忙碌信物，便知道送端故障
        處理：刪除忙碌信物及資料，產生新的自由信物

    多重信物問題

      原因：雜訊干擾
      偵測：送端發現回收之資料SA不是自己
      處理：不再送出自由信物，已造成信物流失再由監督者處理

    最小網路位元長度

      當所有工作站都idle時，只有自由信物在網路上傳送。此時必
      　須保證自由信物可以傳送，所以網路位元長度最少是Token
      　長度。信物監督者必須保持此最小長度(藉由延遲也可以)

    信物監督者當機

      偵測：每一工作站均有Free Token Time-out Timer。當一工作
      　站送出一自由信物後，便開始計時。若時間到了而信物未回
      　來，則表示監督者故障
      處理：偵測到之工作站送出忙碌信物，並在資料欄中填入自
      　己的位址。若此資料繞一圈回來，則可成為監督者。若SA比
      　此工作站大，則放棄競爭，否則再送一次

    決定前一工作站(Predecessor)

      目的：便利找出障礙
      方法：信物監督者週期性的送出「開始點名」控制

        1.訊息(Roll-Call-Pull)
        2.信物監督者的下一工作站設定ARI且拷貝SA為前一工作
        　站
        3.當此工作站得到Free Token時，送出「繼續點名」(Roll-
        　Call-Repeat)給下一個工作站
        4.依此類推直到監督者收到沒有ARI設定之「繼續點名」
        　訊號

    網路故障偵測

      硬體故障

        線路斷裂、傳送器/接收器故障
        偵測：工作站接收器收不到訊號
        處理：送出求救(beacon)訊息(含自己及前一工作站
        　位址)

      環境干擾

        不良傳輸環境
        偵測：工作站所接收之資料老是錯誤
        處理：送出資料時，將SEI設定

    網路故障排除

      工作站與RWC間線路斷裂：RWC自動排除(如lobe bypss)
      　。RWC收到beacon訊號
      RWC與RWC間線路斷裂：網路重組

        利用雙環結構

IEEE 802.5 Token Ring

    運作原理：

    MAC資料格式

      Token:其中 AC 之 Token bit = 1

      資料：

        SD : Starting Delimiter
        AC : Access Control

          ppp : priority of the token
          T : Token bit = 0表示資料
          M :用來避免持續傳送資料或高priority token
          RRR : request priority level of the next token

        FC : Frame Control
        　　　Frame type : MAC or LLC frame
        FCS : checksum
        ED : Ending Delimiter

          I : last or more frame
          E : set if error detected

        FS : Frame Status

          A : destination present C : copied
          1. A = 0 and C = 0 : destination not present or not powerd
          　up.
          2. A = 1 and C = 0 : destination present but frame not
          　accepted.
          3. A = 1 and C = 1 : desitnation present and frame copied.

    優先權

      工作站必須取得一個比所欲傳送的資料的優先等級相等或更
      　低的Token才可傳資料
      可預約Token priority (用RRR field)，但須優先權大於原信物
      　上之預約priority
      Token priority 由原提升工作站還原
